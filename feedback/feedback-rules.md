# 反馈协议

你每交一次作业，我都按这个标准反馈，避免泛泛地说“不错”“继续加油”。

## 你提交时必须包含

```text
课程：
成果：
自评分：
我最不确定的地方：
我希望你重点看：
```

## 我反馈时必须包含

```text
得分：xx/100
是否通关：是/否

做得最好的 1 个点：

最需要改的 2 个点：
1.
2.

下一次只改 1 个点：

重练任务：
```

## 反馈原则

- 每次最多指出 2 个主要问题。
- 不同时要求你改颜色、排版、文案、动效。
- 先修影响最大的东西。
- 如果达到通关线，就进入下一课，不继续原地打磨。

## 常见反馈标签

| 标签 | 含义 |
|---|---|
| 层级问题 | 第一眼不知道看哪里 |
| 对比不足 | 文字和背景分不开 |
| 留白不足 | 页面太挤 |
| 风格不统一 | 元素气质不一致 |
| 可读性问题 | 标题、按钮、正文读不清 |
| 过度设计 | 装饰太多，影响主视觉 |
| 可实现性风险 | 设计看起来好，但网页不好做 |
