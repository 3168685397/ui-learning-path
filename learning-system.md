# 课程系统说明

这套课程借鉴多邻国的设计：小关卡、即时反馈、重复练习、通关线、连续学习。

## 学习单位

| 名称 | 含义 |
|---|---|
| Lesson | 每天一个小关卡 |
| Drill | 重练任务，针对错误补一刀 |
| Boss | 阶段挑战，把多个技能合成一个作品 |
| XP | 学习经验值 |
| Rubric | 评分表 |
| Submission | 你的作业 |

## 一节课的固定结构

每节课必须包含：

1. 今日目标
2. 输入素材
3. 操作步骤
4. 必交成果
5. 通关指标
6. 评分表
7. 常见错误
8. 反馈模板
9. 重练任务

如果一节课没有「必交成果」和「通关指标」，它就不是一节合格的课。

## 案例怎么用

案例不是让你照抄，而是让你训练判断力。

每个案例都要看 4 个问题：

1. 它第一眼让你看哪里？
2. 它用了什么方法制造高级感？
3. 它哪里不能直接照搬？
4. 如果改成你的个人主页，要保留什么、删除什么？

学习顺序：

```text
看 showcase 案例
看 examples 标准作业
做自己的 lesson
提交给 Codex 评分
按反馈重练
```

## 反馈闭环

你每次交作业时，用这个格式：

```text
课程：Lesson 01
成果：截图/链接/文字
自评分：xx/100
我最不确定的地方：
我想让你重点看：
```

我会按这个格式反馈：

```text
得分：xx/100
是否通关：是/否
最好的 1 个点：
最需要改的 2 个点：
下一次只改：
重练任务：
```

## XP 和等级

| 行为 | XP |
|---|---:|
| 完成一节课 | 10 |
| 达到 85 分以上 | +5 |
| 完成重练 | +5 |
| 完成 Boss 关 | 30 |
| 连续 3 天 | +15 |
| 连续 7 天 | +40 |

## 不追求完美

每节课只改一个核心能力：

- 今天练观察，就不要急着做设计。
- 今天练布局，就不要纠结动效。
- 今天练动效，就不要大改文案。

这样进步会更稳。
